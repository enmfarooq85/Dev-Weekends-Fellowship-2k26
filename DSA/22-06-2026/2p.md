# What is a Pointer?

A pointer is a variable that stores the memory address of another variable.

Instead of storing data directly, it stores the address where the data is located.

```cpp
int x = 10;
int* ptr = &x;
```

Memory Representation

```
Variable     Value      Address

x            10         1000

ptr          1000       2000
```

Here,

* `x` stores 10
* `ptr` stores the address of `x`

---

# Why Do We Need Pointers?

Pointers are used for:

* Dynamic Memory Allocation
* Arrays
* Linked Lists
* Trees
* Graphs
* Passing Large Objects Efficiently
* Operating System Programming
* Game Development
* Memory Management
* Implementing Data Structures

# Memory Basics

Computer memory consists of addresses.

```
Address      Value

1000         10
1004         20
1008         30
1012         40
```

Variables are stored at different addresses.

Pointers help us access those addresses.

---

# Declaring a Pointer

```cpp
int* ptr;
```

or

```cpp
int *ptr;
```

Both are correct.

---

# Initializing a Pointer

```cpp
int x = 100;

int* ptr = &x;
```

`ptr` now stores the address of `x`.

---

# Address-of Operator (`&`)

Returns the memory address.

```cpp
int x = 5;

cout << &x;
```

Output

```
0x61ff08
```

---

# Dereference Operator (`*`)

Accesses the value stored at the address.

```cpp
int x = 10;

int* ptr = &x;

cout << *ptr;
```

Output

```
10
```

---

# Modifying Value Through Pointer

```cpp
int x = 10;

int* ptr = &x;

*ptr = 50;

cout << x;
```

Output

```
50
```

---

# Pointer Arithmetic

```cpp
int arr[] = {10,20,30,40};

int* ptr = arr;

ptr++;
```

Now pointer points to

```
20
```

Similarly,

```cpp
ptr--;
```

moves one element backward.

---

# Null Pointer

A pointer that points to nothing.

```cpp
int* ptr = nullptr;
```

Never dereference a null pointer.

---

# Void Pointer

Can point to any data type.

```cpp
int x = 5;

void* ptr = &x;
```

Needs type casting before dereferencing.

```cpp
cout << *(int*)ptr;
```

---

# Wild Pointer

An uninitialized pointer.

```cpp
int* ptr;
```

Contains garbage address.

Never use it.

---

# Dangling Pointer

Points to memory that has already been freed.

```cpp
int* ptr = new int(10);

delete ptr;
```

Now

```cpp
ptr
```

is dangling.

Good practice

```cpp
ptr = nullptr;
```

---

# Pointer to Pointer (Double Pointer)

Stores the address of another pointer.

```cpp
int x = 10;

int* p = &x;

int** pp = &p;
```

Memory

```
pp
 ↓
 p
 ↓
 x
```

---

# Double Dereferencing

```cpp
cout << **pp;
```

Output

```
10
```

---

# Triple Pointer

```cpp
int*** ppp = &pp;
```

Access value

```cpp
***ppp;
```

---

# Pointers and Arrays

Array name itself is a pointer.

```cpp
int arr[5]={1,2,3,4,5};

cout<<arr;
```

Same as

```cpp
cout<<&arr[0];
```

---

Access using pointer

```cpp
int* ptr = arr;

cout<<*(ptr+2);
```

Output

```
3
```

---

# Pointers and Strings

```cpp
char str[] = "Hello";

char* ptr = str;
```

Move

```cpp
ptr++;
```

Now points to

```
e
```

# Passing Pointer to Function

```cpp
void increment(int* x)
{
    (*x)++;
}

int a = 5;

increment(&a);
```

Output

```
6
```

# Constant Pointer

Cannot change address.

```cpp
int x=10;
int y=20;

int* const ptr=&x;
```

Allowed

```cpp
*ptr=30;
```

Not allowed

```cpp
ptr=&y;
```

---

# Pointer to Constant

Cannot modify value.

```cpp
const int* ptr=&x;
```

Allowed

```cpp
ptr=&y;
```

Not allowed

```cpp
*ptr=100;
```


## shared_ptr

## weak_ptr

# Two Pointer Technique

Two pointers traverse data together.

Example

Reverse Array

```cpp
left = 0;
right = n-1;

while(left<right)
{
    swap(arr[left],arr[right]);

    left++;
    right--;
}
```

Common Problems

* Two Sum II
* Reverse String
* Sort Colors
* Move Zeroes
* Merge Sorted Array
* Valid Palindrome
* Container With Most Water

Complexity

```
Time : O(n)
Space : O(1)
```

---

# Three Pointer Technique

Three pointers solve more complex problems.

Example

Dutch National Flag

```cpp
low = 0;
mid = 0;
high = n-1;
```

Problems

* Sort Colors
* Merge Sorted Arrays
* Linked List Reordering

---

# Fast & Slow Pointer Pattern

Also called Floyd Cycle Detection.

```cpp
slow=head;

fast=head;

while(fast && fast->next)
{
    slow=slow->next;

    fast=fast->next->next;
}
```

Applications

* Detect Cycle
* Find Middle Node
* Happy Number
* Find Start of Cycle
* Palindrome Linked List

Complexity

```
O(n)
```

---

# Sliding Window (Two Pointers)

```cpp
left=0;

for(right=0;right<n;right++)
{
    ...
}
```

Problems

* Longest Substring Without Repeating Characters
* Minimum Window Substring
* Maximum Consecutive Ones
* Fruits Into Basket

# Common Mistakes

❌ Dereferencing a null pointer

```cpp
int* ptr = nullptr;

cout << *ptr;
```

---

❌ Forgetting to free memory

```cpp
int* ptr = new int(10);
```

Missing

```cpp
delete ptr;
```

---

❌ Using dangling pointer

```cpp
delete ptr;

cout << *ptr;
```

---

❌ Uninitialized pointer

```cpp
int* ptr;
```

Always initialize.

# Key Takeaways

* A pointer stores the address of another variable.
* `&` gives the address of a variable.
* `*` dereferences a pointer to access or modify the value.
* Double pointers store the address of another pointer.
* Pointer arithmetic is mainly used with arrays.
* Always initialize pointers to avoid undefined behavior.
* Use `nullptr` instead of `NULL` in modern C++.
* Prefer smart pointers (`unique_ptr`, `shared_ptr`) over raw pointers in modern C++.
* Master two-pointer, fast-slow pointer, sliding window, and three-pointer techniques—they are among the most common coding interview patterns.

# Hash Functions and Hash Maps in C++

## Introduction

A Hash Function is a special function that converts a key (such as a string, integer, or object) into a numerical value called a **hash code**. This hash code is then used to determine where the data should be stored in a hash table.

Hashing allows data to be inserted, searched, and deleted very efficiently, making it one of the most important concepts in Data Structures.

---

# Why Do We Need Hashing?

Suppose we want to store student records.

Without hashing:

* We may need to search through the entire list.
* Search complexity can become O(n).

With hashing:

* We can directly locate the storage position.
* Search complexity becomes O(1) on average.

Example:

```text
Student Name → Roll Number

Farooq → 101
Ahsan  → 102
Hadi   → 103
```

A hash function converts the name into an index:

```text
Hash("Farooq") = 5
Hash("Ahsan")  = 8
Hash("Hadi")   = 2
```

The values are stored at those locations.

---

# What is a Hash Table?

A Hash Table is a data structure that stores key-value pairs using hashing.

Example:

```text
Index   Data

0
1
2       Hadi
3
4
5       Farooq
6
7
8       Ahsan
9
```

The hash function decides the index where the data should be stored.

---

# Working of Hashing

The hashing process consists of two steps:

## Step 1: Generate Hash Code

The key is passed to a hash function.

Example:

```text
Key = "Farooq"

Hash Function
     ↓

Hash Code = 123456
```

## Step 2: Compression Function

The hash code may be very large.

We use a compression function to convert it into a valid array index.

Example:

```text
Hash Code = 123456

123456 % 10

Index = 6
```

The data is stored at index 6.

---

# Components of Hashing

## 1. Key

The value used to access data.

Examples:

```text
Farooq
Ahsan
Student ID
Email
```

## 2. Hash Function

Converts a key into a numerical value.

Example:

```text
Hash("Farooq") = 123456
```

## 3. Hash Code

The numerical output generated by the hash function.

```text
123456
```

## 4. Compression Function

Converts the hash code into a valid table index.

```text
123456 % 10 = 6
```

## 5. Bucket

The storage location where data is stored.

```text
Bucket 6 → Farooq
```

---

# Characteristics of a Good Hash Function

A good hash function should:

## Fast

It should compute quickly.

## Deterministic

The same key should always generate the same hash value.

Example:

```text
Hash("Farooq") = 123456
```

Every time.

## Uniform Distribution

Values should be distributed evenly across buckets.

Bad:

```text
1, 1, 1, 1, 1
```

Good:

```text
1, 5, 8, 2, 9
```

## Minimize Collisions

Different keys should rarely map to the same location.

---

# Collision

A collision occurs when two different keys generate the same index.

Example:

```text
Hash("abc") = 5
Hash("xyz") = 5
```

Both keys want to occupy index 5.

This situation is called a collision.

---

# Collision Handling Techniques

## 1. Closed Addressing (Separate Chaining)

Each bucket stores a linked list.

Example:

```text
Bucket 5

abc → xyz → hello
```

Advantages:

* Easy to implement
* Handles many collisions

Disadvantages:

* Extra memory required

---

## 2. Open Addressing

Instead of using a linked list, find another empty slot.

### Linear Probing

```text
Index 5 occupied

Try 6
Try 7
Store here
```

### Quadratic Probing

```text
5 + 1²
5 + 2²
5 + 3²
```

### Double Hashing

Uses another hash function to find the next location.

Advantages:

* No extra linked list needed

Disadvantages:

* Performance decreases when the table becomes full

---

# Load Factor

Load Factor indicates how full the hash table is.

Formula:

```text
Load Factor = Number of Elements / Number of Buckets
```

Example:

```text
Elements = 8
Buckets = 10

Load Factor = 8/10 = 0.8
```

Higher load factor means more collisions.

---

# Rehashing

When the load factor becomes too high:

1. Create a larger hash table.
2. Increase bucket count.
3. Recalculate all indexes.
4. Move elements into new positions.

This process is called Rehashing.

Example:

```text
Old Buckets = 10

New Buckets = 20
```

All elements are inserted again using the new bucket size.

# C++ STL Hash Map

C++ provides:

## unordered_map

Header:

```cpp
#include <unordered_map>
```

Example:

```cpp
unordered_map<string, int> students;

students["Farooq"] = 101;
students["Ahsan"] = 102;

cout << students["Farooq"];
```

Average Complexity:

| Operation | Complexity |
| --------- | ---------- |
| Insert    | O(1)       |
| Search    | O(1)       |
| Delete    | O(1)       |

Worst Case:

| Operation | Complexity |
| --------- | ---------- |
| Insert    | O(n)       |
| Search    | O(n)       |
| Delete    | O(n)       |

---

# map vs unordered_map

| Feature            | map            | unordered_map |
| ------------------ | -------------- | ------------- |
| Internal Structure | Red-Black Tree | Hash Table    |
| Ordering           | Sorted         | Unsorted      |
| Search             | O(log n)       | O(1) Average  |
| Insert             | O(log n)       | O(1) Average  |
| Delete             | O(log n)       | O(1) Average  |

Use `map` when sorted order is required.

Use `unordered_map` when maximum performance is required.

# Summary

* Hashing converts keys into indexes.
* Hash Functions generate hash codes.
* Hash Tables store data using those indexes.
* Collisions occur when multiple keys map to the same location.
* Separate Chaining and Open Addressing handle collisions.
* Load Factor measures table fullness.
* Rehashing improves performance when the table becomes crowded.
* `unordered_map` in C++ uses hashing to provide O(1) average-time operations.
